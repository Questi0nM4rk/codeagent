# Quality Gates Skill

How code quality is enforced in this ecosystem. All enforcement lives in **ai-guardrails**, not in individual projects.

## The Iron Law

```
ai-guardrails owns ALL quality enforcement.
Projects consume it via `ai-guardrails-init`.
Never hack quality configs directly in a project.
```

## Architecture

```
ai-guardrails repo (upstream)
  ├── configs/          → base templates (ruff.toml, biome.json, etc.)
  ├── templates/
  │   ├── pre-commit/   → hook definitions per language
  │   └── workflows/    → CI templates (check.yml)
  └── lib/hooks/        → format-and-stage, detect-suppression-comments
        │
        ▼  ai-guardrails-init --force --ci
        │
project repo (consumer)
  ├── .guardrails-exceptions.toml  (SOURCE OF TRUTH - owner edits)
  ├── ruff.toml                    (AUTO-GENERATED by codeagent generate-configs)
  ├── biome.json                   (AUTO-GENERATED)
  ├── .markdownlint.jsonc          (AUTO-GENERATED)
  ├── .codespellrc                 (AUTO-GENERATED)
  ├── .suppression-allowlist       (AUTO-GENERATED)
  ├── .pre-commit-config.yaml      (generated by ai-guardrails-init)
  └── .github/workflows/check.yml  (installed)
```

## Unified Exception Registry

`.guardrails-exceptions.toml` is the SINGLE SOURCE OF TRUTH for ALL exceptions.

```bash
# Validate the registry
codeagent generate-configs --dry-run

# Regenerate all tool configs from registry
codeagent generate-configs

# Never edit generated files directly - edit the registry instead
```

AI agents MUST NOT modify `.guardrails-exceptions.toml`. Ask the project owner.

The registry contains:
- `[global.ruff]` - globally disabled ruff rules with reasons
- `[global.markdownlint]` - disabled markdownlint rules
- `[global.codespell]` - skip patterns and ignore words
- `[global.pyright]` - pyright config overrides
- `[[file_exceptions]]` - per-file-pattern rule exceptions
- `[[inline_suppressions]]` - approved inline comment patterns
- `[skip]` - tool-specific path exclusions

## Adding or Changing Quality Rules

1. **Edit in ai-guardrails repo** (never in the consumer project)
2. Test: `bats tests/bats/` and `pytest tests/`
3. Commit, push, PR in ai-guardrails
4. In consumer project: `ai-guardrails-init --force --ci`
5. Verify: `pre-commit run --all-files`

## Pre-commit Hooks (Execution Order)

```
format-and-stage  → auto-format + re-stage (local only, skipped in CI)
gitleaks          → secret detection
detect-secrets    → enhanced secret scanning
semgrep           → SAST security scan
ruff              → Python lint (ALL rules, --no-fix in CI)
ruff-format       → Python format check
bandit            → Python security
vulture           → Python dead code
biome             → TS/JS lint+format (if detected)
shellcheck        → Shell lint (if detected)
shfmt             → Shell format (if detected)
markdownlint      → Markdown lint
trailing-whitespace, end-of-file, merge-conflicts, large-files
```

## CI Pipeline (check.yml)

Two jobs run in parallel:

### lint

- Pre-commit all hooks (skip format-and-stage, semgrep)
- Semgrep separately (`--config auto --error`)

### test

- `uv sync --all-extras`
- `uv run pytest tests/ --cov=src --cov-branch --cov-fail-under=85`
- `uv run pyright src/`

## Ruff Configuration Highlights

- `select = ["ALL"]` — every rule enabled, disable explicitly
- `line-length = 88`
- `target-version = "py311"`
- `from __future__ import annotations` required (`I002`)
- Key per-file-ignores: tests get relaxed `S101`, `PLR2004`, `ARG`
- `__init__.py` allows unused imports (`F401`)

## Common Commands

```bash
# Run all checks locally
pre-commit run --all-files

# Run specific hook
pre-commit run ruff --all-files

# Python lint + format
uv run ruff check src/ tests/ --fix
uv run ruff format src/ tests/

# Type check
uv run pyright src/
uv run mypy src/ --strict

# Test with coverage
uv run pytest tests/ --cov=src --cov-fail-under=85
```

## When Pre-commit Fails

1. Read the error — it tells you exactly what's wrong
2. Fix in your code (never disable the check)
3. `git add -u` and commit again
4. If a rule is genuinely wrong for your project, fix it **in ai-guardrails**

## Inline Suppressions

`--ignore-noqa` is enabled: inline `# noqa:` comments are documentary only.
Actual suppression is done via per-file-ignores in `ruff.toml` (generated).

Approved inline patterns are listed in `.guardrails-exceptions.toml` under
`[[inline_suppressions]]`. The `detect-suppression-comments` pre-commit hook
blocks any suppression comment not in the generated `.suppression-allowlist`.

To request a new exception: ask the project owner to add it to the registry.
AI agents MUST NOT add inline suppressions or modify the registry.
